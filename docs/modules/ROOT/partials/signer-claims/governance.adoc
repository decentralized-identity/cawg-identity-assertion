=== Named Actor claim governance

_This section is non-normative._

The *<<_named_actor_claim,Named Actor claim>>* mechanism divides governance responsibilities between CAWG and the individual *<<_named_actor_claim_authority,Named Actor claim authorities>>.*

CAWG governs the parts of the mechanism that every validator must understand in common:

* the way trust in an *<<_named_actor_claim_authority,authority>>* is configured by a validator;
* the format used to distribute *<<_named_actor_claim_authority,authority>>* public keys;
* the Verifiable Credential profile described in xref:_named_actor_claim_credential_format[xrefstyle=full]; and
* the syntax of triple `predicate` identifiers used in `credentialSubject.claims`.

CAWG does _not_ standardize the set of possible claim predicates or objects.

Each *<<_named_actor_claim_authority,Named Actor claim authority>>* governs the meaning of the claims it issues:

* the vocabulary of triple predicates and objects within its own namespace;
* the semantics of each claim;
* the policies under which it issues claims.

The result is that a validator only needs to understand Verifiable Credentials, COSE, and the *<<_named_actor_claim_authority_registry,Named Actor claim authority registry>>.* It does not require bespoke code for any particular *<<_named_actor_claim_authority,authority>>* — whether an industry body, a wire service, a government registry, or a future ecosystem. New claims can be introduced as data and configuration rather than as changes to validator software.
