==== KERI signature payload

KERI signature payload is using the `identity.signature` field of the <<CBOR schema>> to store all data that are needed to verify the signature  following `vlei-signature-payload-map` CBOR type as defined below.

Additionally, the fields `sig_type` and `role` of the `identity.signer_payload` map MUST be set as follows:
```CBOR
"sig_type": "cawg.vlei"
"role": "le" / "ecr" /"oor"
```
Where `le`, `ecr` and `oor` values correspond to the the respective vLEI credential type issued to the signing AID(s).

[unnumbered]
===== _vlei-signature-payload-map_ definition

```CBOR
vlei-signature-payload-map = {
  "signer_aid": tstr
  "qvi_cred": tstr
  "le_cred": tstr
  ?"role_cred" = ("role_auth_cred": tstr, "role_cred": tstr)
  "signer_aid_oobi": tstr
  "kel_seq_num": uint
  ?"anchor_seq_num": uint
  "aid_sig": [+tstr]
}
```
See section <<Generating the KERI signature>> for the description of the rules that MUST be followed when creating the payload.
